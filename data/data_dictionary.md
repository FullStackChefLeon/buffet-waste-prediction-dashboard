# Data Dictionary

| Field | Description |
|---|---|
| date | Date of the buffet service record |
| weekday | Day of the week |
| meal_period | Breakfast, Lunch, or Dinner |
| dish_name | Name of the buffet item |
| dish_category | Category of the dish |
| kitchen_section | Kitchen section responsible for the item |
| estimated_guests | Estimated number of guests before service |
| actual_guests | Actual number of guests during service |
| event_type | Type of service: normal, group tour, banquet, or wedding |
| occupancy_rate | Hotel occupancy rate |
| weather | Weather condition |
| prepared_qty | Quantity prepared before or during service |
| leftover_qty | Quantity left over after service |
| leftover_ratio | leftover quantity divided by prepared quantity |
| service_time_remaining | Minutes remaining in buffet service |
| refill_count | Number of refills during service |
| waste_level | Low, Medium, or High waste level |
| possible_reason | Possible reason for leftover pattern |
| recommended_action | Suggested operational adjustment |

## Note

The dataset is synthetic and created for academic portfolio purposes. The field structure is based on a realistic buffet leftover monitoring framework developed from a resort hotel kitchen internship scenario.
